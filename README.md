# YouTube DS

Repository structure aligned to the implementation plan.

## Layout

- `notebooks/` for EDA work
- `data/raw/` for source data
- `data/processed/` for cleaned outputs
- `docs/` for deployment and project notes
- `backend/` for the FastAPI service
- `frontend/` for the React app
- `presentation/` for the slide deck PDF
- `images/` for exported figures
